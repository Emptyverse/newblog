前端自动化部署

这里介绍github+Netlify来部署hexo博客

1.部署

1.1项目同步到Github

1.1.1 在Github上新建一个repository

复制你刚刚新建的repository的地址

git@github.com:Emptyverse/newblog.git

回到项目根目录,将你的本地项目和新建的repository联系起来:

git remote add origin git@github.com:Emptyverse/newblog.git

在当前根目录下新建.gitignore文件 将不需要同步的文件和目录写到.gitignore:

.DS_Store
Thumbs.db
db.json
*.log
node_modules/
themes/
.deploy*/

完成之后，到根目录:

git add .
git commit -m 'commit information'

接着推送到GitHub仓库:

git push --set-upstream origin master

到这里,我们已经讲我们的项目推送到GitHub的master分支下面了。

接下来我们要对hexo进行一些配置:

打开hexo根目录的_config.yml文件找到deploy项，修改成下面这样

deploy:
  type: git #部署方式
  repository: git@github.com:Emptyverse/newblog.git #关联自己的github仓库
  branch: run-page #部署分支

在这里,我们将在这个项目仓库下新建一个run-page分支,至于有什么用，我等一下会解释,先跟着我操作起来.

配置好了之后,保存退出,我们重新执行一下:

$ hexo clean #清理各种缓存和旧文件
$ hexo g     #生成静态文件

部署前要先安装一个依赖，不然会出错，我们安装一下:

npm install hexo-deployer-git --save

最后,我们将public目录同步到Github:

$ hexo d #部署应用

然后再次执行一下,执行完成我们到Github 会发现我们的项目多了一个run-page,这个分支就是我们后面要用来生成我们到静态页面的。

1.2 托管到Netlify

1.我们先到Netlify官网注册一下账号,因为我们是将项目托管到GitHub的,所以我们选择GitHub登录：

2.进入官网后，点击新建网站，选择import an existing project，

3.可以选择自己代码的仓库，我这里是存在GitHub

4.然后选择你想要部署的项目：

5.进一步配置

6.接着Netlify就会自动帮我们生成网址，我的是修改过的，原来的是unrivaled-brioche-92bdca.netlify.com

直接点击网址地址就可以访问自己的项目了。

7.以后我们写好博客之后,直接执行：

$ hexo clean 
$ hexo g     
$ hexo d
复制代码

我们的个人博客就会自动进行刷新,是不是超厉害！！

1.3修改域名

1.选择Domain settings

2.add custom domain， 填入自己的域名

3.然后在你买域名的地方解析域名，我这是用阿里云的
